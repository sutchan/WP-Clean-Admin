# WP Clean Admin 文档格式规范\r\n\r\n## 1. 概述\r\n\r\n为了确保 WP Clean Admin 项目文档的一致性、可读性和可维护性，特制定本文档格式规范。所有项目文档都应遵循本规范，确保文档风格统一，便于阅读和维护。\r\n\r\n## 2. 基本要求\r\n\r\n### 2.1 编码格式\r\n- 所有文档文件必须使用 **UTF-8 编码，无 BOM**\r\n- 换行符统一使用 **Unix LF**\r\n- 禁止使用 Windows CRLF 换行符\r\n\r\n### 2.2 文件命名\r\n- 文件名使用中文，清晰描述文件内容\r\n- 文件名使用连字符 `-` 分隔单词，例如：`文档格式规范.md`\r\n- 禁止使用空格、下划线或其他特殊字符\r\n- 文件扩展名统一为 `.md`\r\n\r\n### 2.3 目录结构\r\n- 目录名使用中文，清晰描述目录内容\r\n- 目录名格式为：`XX-目录名称`，例如：`01-文档指南`\r\n- 目录编号使用两位数字，从 01 开始\r\n- 禁止使用空格、下划线或其他特殊字符\r\n\r\n## 3. Markdown 语法规范\r\n\r\n### 3.1 标题层级\r\n\r\n#### 3.1.1 标题层级规范\r\n- 一级标题：`#`，用于文档标题\r\n- 二级标题：`##`，用于主要章节\r\n- 三级标题：`###`，用于章节下的小节\r\n- 四级标题：`####`，用于小节下的子小节\r\n- 五级标题：`#####`，用于子小节下的细目\r\n- 六级标题：`######`，用于细目下的具体内容\r\n\r\n#### 3.1.2 标题格式要求\r\n- 标题前后各空一行\r\n- 标题与 `#` 之间有一个空格\r\n- 标题文字首字母大写\r\n- 标题文字不使用标点符号\r\n\r\n**示例：**\r\n```markdown\r\n# 文档标题\r\n\r\n## 主要章节\r\n\r\n### 小节标题\r\n\r\n#### 子小节标题\r\n```\r\n\r\n### 3.2 段落\r\n\r\n- 段落之间空一行\r\n- 每行不超过 80 个字符（便于阅读和版本控制）\r\n- 禁止使用连续空格或制表符缩进\r\n- 段落首行不缩进\r\n\r\n### 3.3 代码块\r\n\r\n#### 3.3.1 行内代码\r\n- 使用反引号 `` ` `` 包裹行内代码\r\n- 代码与反引号之间无空格\r\n\r\n**示例：**\r\n```markdown\r\n使用 `wpca_get_settings()` 函数获取插件设置。\r\n```\r\n\r\n#### 3.3.2 代码块\r\n- 使用三个反引号 ``` 包裹代码块\r\n- 代码块前空一行\r\n- 代码块后空一行\r\n- 指定代码语言，便于语法高亮\r\n- 代码缩进使用 4 个空格\r\n- 代码行长度不超过 80 个字符\r\n\r\n**示例：**\r\n```markdown\r\n```php\r\n// 获取插件设置\r\nfunction wpca_get_settings( $key = '', $default = false ) {\r\n    $settings = get_option( 'wpca_settings', array() );\r\n    \r\n    if ( empty( $key ) ) {\r\n        return $settings;\r\n    }\r\n    \r\n    return isset( $settings[$key] ) ? $settings[$key] : $default;\r\n}\r\n```\r\n```\r\n\r\n### 3.4 列表\r\n\r\n#### 3.4.1 无序列表\r\n- 使用 `-` 作为无序列表标记\r\n- 列表标记与内容之间有一个空格\r\n- 列表项之间空一行\r\n- 嵌套列表使用 4 个空格缩进\r\n\r\n**示例：**\r\n```markdown\r\n- 功能一\r\n  - 子功能一\r\n  - 子功能二\r\n- 功能二\r\n- 功能三\r\n```\r\n\r\n#### 3.4.2 有序列表\r\n- 使用数字 `.` 作为有序列表标记\r\n- 列表标记与内容之间有一个空格\r\n- 列表项之间空一行\r\n- 嵌套列表使用 4 个空格缩进\r\n\r\n**示例：**\r\n```markdown\r\n1. 第一步\r\n   1. 子步骤一\r\n   2. 子步骤二\r\n2. 第二步\r\n3. 第三步\r\n```\r\n\r\n### 3.5 链接\r\n\r\n#### 3.5.1 行内链接\r\n- 格式：`[链接文本](链接地址 "可选标题")`\r\n- 链接文本清晰描述链接内容\r\n- 链接地址使用完整 URL\r\n- 禁止使用缩写 URL\r\n\r\n**示例：**\r\n```markdown\r\n[WP Clean Admin GitHub 仓库](https://github.com/sutchan/WPCleanAdmin "WP Clean Admin GitHub 仓库")\r\n```\r\n\r\n#### 3.5.2 参考链接\r\n- 适合多次引用同一链接\r\n- 格式：`[链接文本][链接标识符]`\r\n- 链接定义：`[链接标识符]: 链接地址 "可选标题"`\r\n- 链接定义放在文档末尾\r\n\r\n**示例：**\r\n```markdown\r\n[WP Clean Admin][1] 是一个功能强大的 WordPress 后台管理和优化插件。\r\n\r\n[1]: https://github.com/sutchan/WPCleanAdmin "WP Clean Admin GitHub 仓库"\r\n```\r\n\r\n### 3.6 图片\r\n\r\n#### 3.6.1 图片格式\r\n- 格式：`![图片描述](图片地址 "可选标题")`\r\n- 图片描述清晰准确\r\n- 图片地址使用相对路径或完整 URL\r\n- 禁止使用缩写 URL\r\n\r\n**示例：**\r\n```markdown\r\n![WP Clean Admin 插件截图](images/wpca-screenshot.png "WP Clean Admin 插件截图")\r\n```\r\n\r\n#### 3.6.2 图片尺寸\r\n- 图片宽度不超过 800px\r\n- 图片高度自适应\r\n- 禁止使用过大的图片\r\n\r\n### 3.7 表格\r\n\r\n#### 3.7.1 表格格式\r\n- 表格前后各空一行\r\n- 表头与内容使用 `-` 分隔\r\n- 表格对齐使用 `:` 标记\r\n- 表格内容居中对齐（默认）\r\n- 表格列宽自适应\r\n\r\n**示例：**\r\n```markdown\r\n| 功能模块 | 描述 | 优先级 |\r\n|----------|------|--------|\r\n| 后台清理 | 清理 WordPress 后台冗余内容 | 高 |\r\n| 性能优化 | 提高网站加载速度 | 高 |\r\n| 安全增强 | 增强网站安全性 | 中 |\r\n```\r\n\r\n#### 3.7.2 表格对齐\r\n- 左对齐：`| 左对齐 |`\r\n- 居中对齐：`| :--- |`\r\n- 右对齐：`| ---: |`\r\n\r\n**示例：**\r\n```markdown\r\n| 左对齐 | 居中对齐 | 右对齐 |\r\n| :----- | :------: | -----: |\r\n| 内容1  | 内容2    | 内容3  |\r\n| 内容4  | 内容5    | 内容6  |\r\n```\r\n\r\n### 3.8 强调\r\n\r\n#### 3.8.1 粗体\r\n- 使用 `**` 包裹粗体文本\r\n- 粗体与 `**` 之间无空格\r\n- 用于强调重要内容\r\n\r\n**示例：**\r\n```markdown\r\n**重要提示：** 请确保备份数据库后再进行操作。\r\n```\r\n\r\n#### 3.8.2 斜体\r\n- 使用 `*` 包裹斜体文本\r\n- 斜体与 `*` 之间无空格\r\n- 用于强调或引用\r\n\r\n**示例：**\r\n```markdown\r\n*注意：* 此功能仅适用于 WordPress 5.0+ 版本。\r\n```\r\n\r\n#### 3.8.3 删除线\r\n- 使用 `~~` 包裹删除线文本\r\n- 删除线与 `~~` 之间无空格\r\n- 用于表示已删除或过时的内容\r\n\r\n**示例：**\r\n```markdown\r\n~~此功能已废弃~~\r\n```\r\n\r\n### 3.9 引用\r\n\r\n- 使用 `>` 作为引用标记\r\n- 引用标记与内容之间有一个空格\r\n- 引用前后各空一行\r\n- 嵌套引用使用多个 `>` 标记\r\n\r\n**示例：**\r\n```markdown\r\n> 这是一个引用。\r\n> \r\n> > 这是一个嵌套引用。\r\n```\r\n\r\n### 3.10 注释\r\n\r\n- 使用 HTML 注释格式：`<!-- 注释内容 -->`\r\n- 注释内容清晰描述注释目的\r\n- 禁止使用过多注释\r\n- 注释放在适当位置，不影响文档可读性\r\n\r\n**示例：**\r\n```markdown\r\n<!-- 以下是功能模块列表 -->\r\n- 功能一\r\n- 功能二\r\n- 功能三\r\n```\r\n\r\n## 4. 文档内容规范\r\n\r\n### 4.1 内容组织\r\n\r\n- 文档内容逻辑清晰，结构合理\r\n- 内容由浅入深，循序渐进\r\n- 重点内容突出显示\r\n- 避免冗余内容\r\n\r\n### 4.2 语言规范\r\n\r\n- 使用简洁、准确的中文\r\n- 避免使用口语化表达\r\n- 避免使用模糊、歧义的词汇\r\n- 术语使用一致\r\n- 代码示例使用英文注释\r\n\r\n### 4.3 代码示例\r\n\r\n- 代码示例完整、可运行\r\n- 代码示例包含必要的注释\r\n- 代码示例遵循项目代码规范\r\n- 代码示例使用真实的函数和类名\r\n- 代码示例格式正确，缩进一致\r\n\r\n**示例：**\r\n```php\r\n// 获取插件版本\r\nfunction wpca_get_version() {\r\n    $plugin_data = get_plugin_data( WPCA_PLUGIN_FILE );\r\n    return $plugin_data['Version'];\r\n}\r\n```\r\n\r\n### 4.4 术语表\r\n\r\n- 文档中使用的术语应在文档开头或末尾定义\r\n- 术语定义清晰准确\r\n- 术语使用一致\r\n\r\n**示例：**\r\n\r\n## 术语表\r\n\r\n| 术语 | 定义 |\r\n|------|------|\r\n| WP Clean Admin | 一个 WordPress 后台管理和优化插件 |\r\n| BOM | Byte Order Mark，字节顺序标记 |\r\n| LF | Line Feed，换行符 |\r\n| CRLF | Carriage Return Line Feed，回车换行符 |\r\n\r\n## 5. 文档结构规范\r\n\r\n### 5.1 文档结构\r\n\r\n所有文档应包含以下基本结构：\r\n\r\n1. **标题**：文档的主标题\r\n2. **概述**：文档的简要介绍和目的\r\n3. **正文**：文档的主要内容，按章节组织\r\n4. **结论**：文档的总结和结论\r\n5. **参考资料**：文档中引用的外部资源\r\n6. **术语表**：文档中使用的术语定义\r\n\r\n### 5.2 文档模板\r\n\r\n```markdown\r\n# 文档标题\r\n\r\n## 1. 概述\r\n\r\n文档的简要介绍和目的。\r\n\r\n## 2. 章节标题\r\n\r\n章节内容。\r\n\r\n### 2.1 小节标题\r\n\r\n小节内容。\r\n\r\n#### 2.1.1 子小节标题\r\n\r\n子小节内容。\r\n\r\n## 3. 结论\r\n\r\n文档的总结和结论。\r\n\r\n## 4. 参考资料\r\n\r\n- [参考资料1](链接地址)\r\n- [参考资料2](链接地址)\r\n\r\n## 5. 术语表\r\n\r\n| 术语 | 定义 |\r\n|------|------|\r\n| 术语1 | 定义1 |\r\n| 术语2 | 定义2 |\r\n```\r\n\r\n## 6. 文档审核规范\r\n\r\n### 6.1 审核内容\r\n\r\n- 文档格式是否符合规范\r\n- 文档内容是否完整、准确\r\n- 文档语言是否简洁、准确\r\n- 代码示例是否完整、可运行\r\n- 术语使用是否一致\r\n\r\n### 6.2 审核流程\r\n\r\n1. 文档编写完成后，提交审核\r\n2. 审核人员检查文档格式和内容\r\n3. 审核人员提出修改意见\r\n4. 文档编写人员修改文档\r\n5. 再次提交审核，直到通过\r\n\r\n### 6.3 审核标准\r\n\r\n- 文档格式符合本规范\r\n- 文档内容完整、准确\r\n- 文档语言简洁、准确\r\n- 代码示例完整、可运行\r\n- 术语使用一致\r\n- 文档易于理解和维护\r\n\r\n## 7. 文档维护规范\r\n\r\n### 7.1 文档更新\r\n\r\n- 文档内容应及时更新，确保与项目实际情况一致\r\n- 文档更新后，应在文档末尾添加更新日志\r\n- 更新日志应包含更新时间、更新内容和更新人员\r\n\r\n### 7.2 文档版本控制\r\n\r\n- 文档应使用版本控制工具管理\r\n- 文档版本号遵循语义化版本规范（MAJOR.MINOR.PATCH）\r\n- 文档版本号与项目版本号保持一致\r\n\r\n### 7.3 文档归档\r\n\r\n- 过时的文档应归档保存\r\n- 归档文档应标记为 "已归档"\r\n- 归档文档应保留在项目中，便于查阅历史版本\r\n\r\n## 8. 总结\r\n\r\n本规范旨在确保 WP Clean Admin 项目文档的一致性、可读性和可维护性。所有项目文档都应遵循本规范，确保文档风格统一，便于阅读和维护。\r\n\r\n通过遵循本规范，我们可以：\r\n- 提高文档的可读性和可维护性\r\n- 确保文档内容的一致性和准确性\r\n- 便于团队成员协作和沟通\r\n- 提高项目的整体质量\r\n\r\n希望所有团队成员都能严格遵守本规范，共同维护高质量的项目文档。
